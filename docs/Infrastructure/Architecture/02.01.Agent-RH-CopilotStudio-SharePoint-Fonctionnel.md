## 1.2 Cas d'utilisation

### UC-001 : Poser une question RH à l'agent

**Acteur principal :** Employé Sani Marc
**Objectif :** Obtenir une réponse ancrée dans les documents RH officiels via le chatbot Teams sans solliciter l'équipe RH.
**Préconditions :**
- L'employé est authentifié dans Microsoft Teams via Azure AD (SSO)
- L'application Teams « Agent RH Sani Marc » est déployée et accessible pour l'employé
- Le corpus documentaire SharePoint contient au moins un document indexé

**Scénario nominal :**
1. L'employé ouvre le canal/conversation « Agent RH Sani Marc » dans Teams.
2. L'agent affiche un message d'accueil indiquant qu'il s'agit d'un agent IA (non humain) et propose des sujets fréquents en Adaptive Cards.
3. L'employé saisit sa question en langage naturel (ex. : « Combien de jours de vacances ai-je droit après 5 ans de service ? »).
4. Copilot Studio classifie l'intention via le moteur NLU et identifie le topic correspondant.
5. Le topic déclenche une requête vers Azure AI Search avec le texte de la question (embedding vectoriel + full-text).
6. Azure AI Search applique le security trimming ACL : seuls les documents accessibles à l'identité de l'employé entrent dans les résultats.
7. Les chunks pertinents sont transmis à Azure OpenAI GPT-4o comme contexte.
8. GPT-4o génère une réponse en langage naturel ancrée dans les documents récupérés.
9. Copilot Studio retourne la réponse à l'employé avec les citations sources (nom du document + lien SharePoint si accessible).
10. L'employé évalue la réponse via les boutons de feedback (pouce haut / pouce bas).

**Scénarios alternatifs :**
- *Aucun document pertinent trouvé dans le corpus :* L'agent répond qu'il ne dispose pas de l'information et propose d'escalader vers un agent RH humain.
- *Score de confiance de l'agent inférieur au seuil configuré (< 60 %) :* L'agent informe l'employé de son incertitude, présente sa meilleure réponse partielle et propose l'escalade.
- *Question hors périmètre RH (ex. : question sur la paie ou les congés individuels) :* L'agent détecte le hors-périmètre via un topic dédié et redirige vers le SIRH ou l'équipe RH.

**Exigences spéciales :** La réponse doit inclure un disclaimer automatique sur les questions relatives aux conventions collectives (« Consultez le texte officiel ou votre représentant syndical pour toute application spécifique »).
**Fréquence :** Estimée à 50–200 questions/jour pour 500 employés.
**Canaux :** Microsoft Teams (principal), SharePoint Web Part (secondaire).

### UC-002 : Dépôt d'un document RH par le responsable RH

**Acteur principal :** Responsable RH
**Objectif :** Mettre à jour le corpus documentaire de l'agent en déposant, modifiant ou retirant un document dans SharePoint, sans intervention TI.
**Préconditions :**
- Le responsable RH dispose des droits d'écriture sur la bibliothèque SharePoint « Politiques-RH »
- Le flux Power Automate d'indexation est actif

**Scénario nominal :**
1. Le responsable RH ouvre la bibliothèque SharePoint « Politiques-RH ».
2. Il dépose un nouveau document (PDF ou Word) dans le dossier de catégorie approprié (ex. : `/Vacances-Conges/`, `/Conventions-Collectives/`).
3. Power Automate détecte l'événement « Fichier créé ou modifié » via le connecteur SharePoint.
4. Le flux appelle le déclencheur d'indexation de l'indexeur Azure AI Search (endpoint REST dédié).
5. L'indexeur Azure AI Search récupère le fichier via Microsoft Graph API, segmente le contenu en chunks (taille configurée : 1 000 tokens, overlap 200 tokens), génère les embeddings via Azure OpenAI et met à jour l'index.
6. Dans un délai de 24 heures maximum, le document est disponible dans les réponses de l'agent.
7. Le responsable RH peut vérifier l'indexation en posant une question de test à l'agent.

**Scénarios alternatifs :**
- *Document en format image scannée (PDF image) :* L'indexeur tente OCR via Azure AI Document Intelligence (si configuré) ; sinon, le document est ignoré et un courriel d'alerte est envoyé à l'administrateur TI.
- *Erreur de flux Power Automate :* Un courriel d'alerte est envoyé à l'administrateur TI via le connecteur Office 365 Outlook ; l'administrateur peut déclencher l'indexation manuellement depuis le portail Azure AI Search.
- *Document retiré de SharePoint :* Power Automate détecte la suppression, déclenche la réindexation différentielle qui supprime le chunk correspondant de l'index.

**Fréquence :** Hebdomadaire (mises à jour courantes) ou à la demande (nouvelles politiques).
**Canaux :** SharePoint Online (bibliothèque Politiques-RH).

### UC-003 : Escalade vers un agent RH humain

**Acteur principal :** Employé Sani Marc
**Objectif :** Transférer la conversation à un membre de l'équipe RH lorsque l'agent IA ne peut pas fournir de réponse satisfaisante.
**Préconditions :**
- L'employé est en conversation avec l'agent
- L'escalade est déclenchée manuellement par l'employé ou automatiquement par l'agent (seuil de confiance)

**Scénario nominal :**
1. L'agent détecte un seuil de confiance bas ou l'employé clique sur le bouton « Parler à un agent RH ».
2. L'agent collecte un résumé automatique de la conversation (3-5 phrases) et l'intention non résolue.
3. L'agent envoie une Adaptive Card de confirmation à l'employé : « Je vous transfère à l'équipe RH. Un agent vous répondra sous [délai]. La conversation sera partagée avec eux. »
4. Une notification Teams est envoyée au canal privé « RH-Escalades » avec le résumé de conversation et les coordonnées de l'employé.
5. L'employé reçoit une confirmation de prise en charge avec le délai estimé de réponse.
6. Un membre RH répond directement à l'employé via Teams (fil de conversation ou courriel selon disponibilité).

**Scénarios alternatifs :**
- *Hors des heures ouvrables :* L'agent informe l'employé que l'équipe RH sera disponible à partir de 8 h le prochain jour ouvrable et offre la possibilité de laisser un message.
- *Problème technique avec le canal d'escalade :* L'agent fournit l'adresse courriel de l'équipe RH en remplacement.

**Fréquence :** Estimée à 5–15 escalades/jour (< 20 % des conversations).
**Canaux :** Microsoft Teams → Canal privé RH-Escalades.

### UC-004 : Consultation du tableau de bord analytique

**Acteur principal :** Responsable RH, Administrateur TI
**Objectif :** Consulter les métriques d'utilisation et de qualité de l'agent pour piloter son amélioration continue.
**Préconditions :**
- L'utilisateur dispose du rôle « Administrateur Copilot Studio » ou « Analyste » dans l'environnement Power Platform.

**Scénario nominal :**
1. L'utilisateur se connecte à la console Copilot Studio (make.microsoft.com) avec ses identifiants Azure AD.
2. Il ouvre le tableau de bord « Analytics » de l'agent « Agent RH Sani Marc ».
3. Il consulte les métriques clés : sessions totales, taux d'escalade, taux de satisfaction (feedback), topics les plus déclenchés, taux de résolution.
4. Il filtre par période (7 jours, 30 jours, personnalisé) et par canal (Teams, SharePoint Web Part).
5. Il exporte les données en CSV pour reporting interne si nécessaire.

**Fréquence :** Hebdomadaire (pilotage) et mensuelle (rapport de direction).
**Canaux :** Console Copilot Studio (portail web).

## 1.3 Processus métier

### Processus : Traitement d'une question RH (RAG Question Answering)

| Attribut | Valeur |
|----------|--------|
| **Déclencheur** | Employé saisit un message texte dans le canal Teams de l'agent |
| **Fréquence** | 50–200 fois/jour |
| **Durée moyenne** | 3–4 secondes (latence p50) |
| **Acteurs** | Employé, Copilot Studio, Azure AI Search, Azure OpenAI GPT-4o, Azure AD |

**Flux :**

```text
Employé (Teams)
    │
    │── [1] Message texte saisi ──▶ Copilot Studio (NLU / Classification)
                                          │
                         ┌────────────────┴────────────────┐
                [Topic identifié]                  [Topic non identifié]
                         │                                  │
          [2] Topic RH déclenché               [Topic Fallback / Hors périmètre]
                         │                                  │
    [3] Azure AD – vérification SSO token             Réponse générique + escalade
                         │
    [4] Requête Azure AI Search (hybrid vectorielle + full-text + security trimming ACL)
                         │
             ┌───────────┴───────────┐
      [Chunks pertinents trouvés]  [Aucun résultat]
             │                           │
    [5] Augmentation du prompt      Message « non disponible »
        (GPT-4o avec contexte)      + proposition escalade
             │
    [6] Azure OpenAI GPT-4o génère la réponse ancrée
             │
    [7] Post-traitement : ajout disclaimer si convention collective
             │
    [8] Réponse + citations sources ──▶ Employé (Adaptive Card Teams)
             │
    [9] Boutons feedback (pouce haut / pouce bas) envoyés à Copilot Analytics
```

**Étapes détaillées :**
1. **Réception du message** — Input : texte libre employé / Output : intent classifié par NLU Copilot Studio
2. **Classification NLU et routage** — Input : intent + entités / Output : topic déclenché ou fallback activé
3. **Vérification d'identité et ACL** — Input : token Azure AD employé / Output : liste des bibliothèques SharePoint accessibles à cet employé
4. **Recherche hybride Azure AI Search** — Input : texte de la question + identité employée / Output : top-5 chunks pertinents avec scores
5. **Génération de réponse GPT-4o** — Input : prompt système + chunks récupérés + question / Output : réponse en langage naturel avec sources
6. **Post-traitement métier** — Input : réponse brute GPT-4o + catégorie du topic / Output : réponse enrichie (disclaimer convention collective si applicable)
7. **Présentation à l'employé** — Input : réponse enrichie / Output : carte Teams avec texte, citations, boutons feedback

**Scénarios d'exception :**
- Azure AI Search indisponible : l'agent détecte le timeout (> 10 secondes), affiche un message d'erreur générique et propose l'escalade.
- Azure OpenAI GPT-4o en surcharge (HTTP 429) : Copilot Studio applique un retry automatique avec backoff exponentiel (3 tentatives, 2/4/8 secondes) ; si toujours en échec, message d'erreur + escalade.

### Processus : Indexation automatique des documents RH

| Attribut | Valeur |
|----------|--------|
| **Déclencheur** | Dépôt, modification ou suppression d'un fichier dans la bibliothèque SharePoint « Politiques-RH » |
| **Fréquence** | À la demande (événement SharePoint) + indexation complète hebdomadaire (dimanche 2 h HNE) |
| **Acteurs** | Responsable RH, Power Automate, Azure AI Search Indexer, Azure OpenAI Embeddings, Microsoft Graph API |

**Flux :**

```text
Responsable RH
    │
    │── [1] Dépôt / modification / suppression fichier ──▶ SharePoint Online (Bibliothèque Politiques-RH)
                                                                    │
                                               [2] Événement détecté par Power Automate
                                                                    │
                                               [3] Flux Power Automate déclenché
                                                                    │
                                               [4] Appel REST – Azure AI Search Indexer (run indexer)
                                                                    │
                                               [5] Indexeur récupère le fichier via Microsoft Graph API
                                                                    │
                                               [6] Azure AI Document Intelligence (OCR si image)
                                                                    │
                                     ┌─────────────────────────────┘
                                     │
                          [7] Chunking du contenu (1 000 tokens, overlap 200)
                                     │
                          [8] Génération des embeddings (Azure OpenAI text-embedding-3-large)
                                     │
                          [9] Mise à jour de l'index Azure AI Search
                          (champs : id, content, content_vector, source_url, category, permissions)
                                     │
                          [10] Log de confirmation dans Power Automate Run History
```

**Étapes détaillées :**
1. **Dépôt dans SharePoint** — Input : fichier (PDF, DOCX, PPTX) / Output : événement SharePoint OnFileCreated/OnFileModified
2. **Déclenchement Power Automate** — Input : événement SharePoint / Output : appel REST vers Azure AI Search Indexer API
3. **Récupération du fichier** — Input : URL document SharePoint + token Graph API / Output : binaire du fichier
4. **OCR si nécessaire** — Input : image scannée / Output : texte structuré (via Azure AI Document Intelligence)
5. **Chunking** — Input : texte brut / Output : tableau de chunks (1 000 tokens max, overlap 200 tokens)
6. **Génération d'embeddings** — Input : texte du chunk / Output : vecteur 3 072 dimensions (text-embedding-3-large)
7. **Indexation** — Input : chunks + vecteurs + métadonnées + ACL / Output : index Azure AI Search mis à jour

**Scénarios d'exception :**
- Fichier trop volumineux (> 16 Mo) : le flux Power Automate déclenche l'indexeur en mode segmentation par lots et envoie une alerte si l'indexation dépasse 60 minutes.
- Format de fichier non supporté (ex. : `.xls` ancien format, `.msg`) : le flux consigne l'erreur dans SharePoint (colonne « Statut d'indexation ») et notifie l'administrateur TI par courriel.
- Suppression d'un document : l'indexeur différentiel utilise la politique de suppression logique (champ `IsDeleted = true`) puis purge les chunks de l'index dans la prochaine exécution planifiée.

### 1.4.2 Concepts métier

| Concept | Définition | Propriétés clés | Règles associées |
|---------|-----------|----------------|------------------|
| DocumentRH | Fichier officiel RH déposé dans SharePoint — source de vérité pour les réponses de l'agent. | Nom, Catégorie, URL SharePoint, Date de mise à jour, ACL groupes Azure AD, Statut d'indexation | Seuls les documents dans la bibliothèque « Politiques-RH » sont indexés. Un document retiré de SharePoint doit être retiré de l'index dans les 24 h. |
| ChunkDocumentaire | Segment de texte découpé d'un document RH, enrichi d'un vecteur d'embedding, stocké dans l'index Azure AI Search. | Texte (1 000 tokens max), Vecteur 3 072 dimensions, PageSource, Score de pertinence, ACL hérités du document parent | Un chunk hérite des ACL du document parent. Le score > seuil configurable (0.6) détermine si le chunk est utilisé dans la réponse. |
| Conversation | Session d'interaction entre un employé et l'agent, identifiée par un GUID pseudonyme dans les logs. | SessionId (GUID), DateDébut, Canal, Langue, Statut (Active/Terminée/Escaladée) | Les logs de conversation sont conservés 12 mois (PIPEDA). Aucun NIP ou identifiant nominatif dans les logs stockés dans Purview. |
| Topic | Scénario de conversation configuré dans Copilot Studio, déclenché par mots-clés ou intentions NLU. | Nom, Phrases déclencheuses, Seuil de confiance, Type (Général/Hors-périmètre/Escalade/Accueil), Source activée | Un topic doit avoir un seuil de confiance minimum de 60 %. Le topic « Escalade » est déclenché si aucun topic n'atteint 40 % de confiance. |
| Escalade | Transfert d'une conversation de l'agent IA vers un agent RH humain, accompagné du résumé de la conversation. | EscaladeId, SessionId parent, Raison, DateHeure, StatutPEC (En attente / Pris en charge) | Chaque escalade génère une notification Teams dans le canal privé « RH-Escalades ». Le délai de prise en charge cible est de 2 heures ouvrables. |
| CorpusDocumentaire | Ensemble de tous les documents RH indexés, représentant la base de connaissance de l'agent. | Volume de documents, Date dernière indexation complète, Couverture thématique | 100 % des politiques RH actives doivent être indexées avant le go-live. Un document obsolète doit être retiré dans les 48 h de son remplacement. |

## 1.5 Règles métier

| ID | Règle | Criticité | Implémentation |
|----|-------|-----------|----------------|
| RM-001 | L'agent ne retourne jamais un document auquel l'employé n'a pas accès selon ses permissions SharePoint. | Haute | Security trimming ACL dans Azure AI Search — héritage des permissions SharePoint via Microsoft Graph identity |
| RM-002 | Toute réponse incluant un extrait de convention collective doit afficher un disclaimer légal. | Haute | Post-traitement Copilot Studio : détection de la catégorie « Conventions-Collectives » dans les métadonnées du chunk, ajout automatique de la note légale |
| RM-003 | L'employé doit être informé dès le premier message qu'il interagit avec un agent IA et non un humain. | Haute | Topic d'accueil obligatoire avec message explicite et icône robot — impossible de bypasser via configuration Copilot Studio |
| RM-004 | Un seuil de confiance inférieur à 40 % déclenche automatiquement une proposition d'escalade. | Haute | Configuration du seuil de fallback dans Copilot Studio — topic « Fallback » activé si aucun topic ≥ 40 % de confiance |
| RM-005 | Les questions relatives à des données personnelles individuelles (paie, dossier employé, absences personnelles) sont hors périmètre et redirigées vers le SIRH ou la DRH. | Haute | Topic dédié « Hors-Périmètre » — détection par mots-clés (paie, salaire, dossier, absence, personnel) et redirection explicite |
| RM-006 | Les logs de conversation sont conservés 12 mois puis automatiquement supprimés. | Haute | Politique de rétention Microsoft Purview — délai de 12 mois configuré sur le label « Agent-RH-Conversations » |
| RM-007 | Un document déposé dans SharePoint doit être indexé dans un délai maximum de 24 heures. | Moyenne | Flux Power Automate événementiel + alerte si l'indexation dépasse 24 h (monitoring via Power Automate Run History) |
| RM-008 | Aucune donnée personnelle identifiable ne doit être stockée dans les logs de conversation sans consentement explicite. | Haute | GUID pseudonyme en lieu et place de l'identité dans les logs Purview — mapping GUID → identité maintenu séparément avec accès restreint |
| RM-009 | L'agent doit citer la source de chaque affirmation dans sa réponse (nom du document + lien si accessible). | Moyenne | Copilot Studio : configuration du comportement de citation dans les Generative Answers — citations activées par défaut |
| RM-010 | Lors d'une escalade, le résumé de conversation transmis à l'équipe RH doit exclure toute information sensible saisie par l'employé. | Haute | Post-traitement du résumé d'escalade — validation manuelle du topic Escalade avant déploiement production |

# 2. Architecture applicative

> **Note :** La solution repose sur une approche **SaaS low-code** (Microsoft Copilot Studio + Power Platform). Il n'y a donc
> pas de stack frontend/backend traditionnel. Cette section décrit les composants de la plateforme Microsoft qui jouent
> les rôles équivalents.

## 2.1 Composant Interface (équivalent Frontend)

**Plateforme :** Microsoft Teams — Application Agent RH Sani Marc
**Interface secondaire :** SharePoint Web Part (Copilot Studio embed)
**Authentification :** Azure AD SSO natif (OAuth 2.0 / OIDC) — aucune session applicative à gérer

**Composants d'interface utilisés :**

| Composant | Usage | Configuration |
|-----------|-------|---------------|
| Canal Teams | Conversation principale avec l'agent | Déployé via Admin Center Teams — groupe cible : tous les employés Sani Marc |
| Adaptive Cards | Affichage structuré des réponses, boutons d'action (feedback, escalade), menus de navigation | Templates JSON configurés dans Copilot Studio — version Adaptive Cards 1.5 |
| SharePoint Web Part | Interface alternative sur l'intranet RH | Composant natif Copilot Studio embed — authentification SharePoint héritée |
| Welcome Card | Accueil avec sujets fréquents (topics suggérés) | 4-6 boutons de raccourcis configurables dans Copilot Studio (mise à jour sans code) |

**Justification des choix :** Microsoft Teams est déjà déployé sur 100 % des postes Sani Marc. L'approche Web Part SharePoint est un canal secondaire sans surcoût de développement. Les Adaptive Cards offrent une expérience structurée sans développement frontend custom.

## 2.2 Composant Orchestration (équivalent Backend)

**Plateforme :** Microsoft Copilot Studio (Power Platform)
**Environnement :** Power Platform Environment « Sani Marc — Production » (Canada)
**Modèle de licence :** 25 000 sessions/mois incluses dans le plan Microsoft 365; modèle pay-per-session disponible (0,01 $/session) en cas de dépassement.

# 7. Qualités de service — Implémentation

## 7.1 Stratégies d'optimisation de performance

### Configuration Azure AI Search

- **Chunking optimisé :** Taille de 1 000 tokens avec overlap de 200 tokens — compromis entre contexte suffisant et nombre de tokens consommés par GPT-4o
- **Recherche hybride (BM25 + HNSW) :** Pondération `vectorWeight=0.5, textWeight=0.5` en phase pilote ; ajustée après analyse des résultats
- **Re-ranking sémantique :** Activé — améliore de ~20 % la pertinence des top-5 résultats selon les benchmarks Microsoft
- **Replicas Azure AI Search :** 3 replicas en production pour la haute disponibilité des requêtes de recherche
- **Partitions :** 1 partition en phase pilote, passage à 2 partitions si le corpus dépasse 25 Go

### Configuration Copilot Studio

- **Mise en cache des réponses récurrentes :** Copilot Studio met en cache les réponses identiques pour des questions identiques dans la même session (natif)
- **Topics spécialisés en priorité :** Les topics thématiques (Vacances, Conventions, Avantages) ont un seuil plus élevé (70 %) que le topic général (60 %), réduisant les appels AI Search non nécessaires pour les questions hors-périmètre
- **Adaptive Cards asynchrones :** L'affichage de la Welcome Card ne bloque pas le démarrage de la session

### Performance de l'indexation

- **Indexation différentielle** (par défaut) : seuls les documents modifiés sont réindexés, pas le corpus complet
- **Batch d'embeddings :** 16 chunks par appel Azure OpenAI (max API), réduisant de 94 % le nombre d'appels vs. 1 chunk par appel
- **Indexation complète planifiée :** Dimanche 2 h HNE — période de faible utilisation ; durée estimée < 2 heures pour le corpus initial

## 8.2 Validation des inputs

### Couche Copilot Studio (équivalent Frontend + Backend)

- **Longueur maximale du message :** 500 caractères — configuré dans les propriétés de l'agent
- **Caractères de contrôle filtrés :** Copilot Studio normalise les inputs avant traitement NLU
- **Content moderation Azure OpenAI :** Niveau strict — bloque les tentatives de génération de contenu inapproprié ou les instructions malveillantes dans le prompt utilisateur
- **Mots-clés hors-périmètre :** Liste configurée dans le topic « Hors-Périmètre » pour détecter les tentatives d'informations sensibles (paie, données personnelles)

### Protection contre le Prompt Injection

```text
Mécanisme de défense multicouche :

Couche 1 — Prompt système (Copilot Studio) :
  → Instructions explicites de ne répondre qu'à partir du contexte documentaire
  → Interdiction de suivre des instructions saisies par l'utilisateur qui contredisent le prompt système

Couche 2 — Content moderation (Azure OpenAI) :
  → Niveau « strict » — bloque les outputs violant les politiques Microsoft
  → Catégories surveillées : hate, self-harm, violence, sexual content, jailbreak

Couche 3 — Topics Copilot Studio :
  → Détection de patterns de prompt injection connus (« ignore tes instructions », « tu es maintenant... »)
  → Redirection vers le topic Fallback + flag dans les logs analytiques

Couche 4 — Revue périodique des conversations :
  → Revue mensuelle d'un échantillon aléatoire de conversations par l'administrateur TI
  → Recherche de patterns d'abus ou de contournement
```

### Validation des fichiers déposés par la DRH

- **Types MIME autorisés :** `application/pdf`, `application/vnd.openxmlformats-officedocument.wordprocessingml.document`, `application/vnd.openxmlformats-officedocument.presentationml.presentation`
- **Taille maximale :** 100 Mo (limite SharePoint Online)
- **Scan antivirus :** Microsoft Defender for SharePoint — actif par défaut sur le tenant Sani Marc (détection automatique de fichiers malveillants avant indexation)

- **Escalade :** Incident de sécurité → Directeur TI immédiatement ; qualité des réponses → DRH dans les 4 heures

| Engagement hypercare | Délai |
|---------------------|-------|

| Incident de sécurité | Rollback dans les 15 minutes |
| Demande d'ajout/retrait de document | Indexé dans les 24 heures |

**Standup quotidien hypercare :** 9 h HNE — 15 minutes — dans le canal `#RH-Agent-Hypercare`
- Métriques de la veille (sessions, CSAT, escalades, incidents)
- Questions de qualité signalées par la DRH ou les employés
- Actions correctives planifiées

- Vérification quotidienne du dashboard Copilot Studio Analytics

## Annexe A — Glossaire technique

| Terme | Définition | Contexte |
|-------|-----------|---------|
| RAG (Retrieval-Augmented Generation) | Technique d'IA combinant la recherche de documents pertinents avec la génération de texte. Les documents récupérés servent de contexte au modèle de langage pour ancrer ses réponses. | Architecture de base de l'Agent RH — garantit que les réponses sont basées sur les politiques officielles, pas sur les connaissances générales du LLM |
| Chunk | Fragment de texte extrait d'un document lors de l'indexation (1 000 tokens max dans notre configuration). Chaque document est découpé en plusieurs chunks qui sont indexés individuellement. | Unité de base de l'index Azure AI Search — les citations sources dans les réponses pointent vers le document parent du chunk |
| Embedding | Représentation vectorielle numérique d'un texte en N dimensions. Les textes sémantiquement similaires ont des vecteurs proches dans l'espace vectoriel. | Généré par `text-embedding-3-large` (3 072 dimensions) — utilisé pour la recherche sémantique hybride |
| Security Trimming | Mécanisme d'Azure AI Search filtrant les résultats de recherche selon les permissions de l'utilisateur authentifié, héritées des ACL SharePoint. | Non négociable — garantit qu'un employé ne reçoit jamais une réponse basée sur un document auquel il n'a pas accès |
| Copilot Studio | Outil low-code Microsoft pour créer des agents conversationnels IA (chatbots). Orchestre les topics, le NLU, les connexions aux sources de données et la génération de réponses. | Cerveau de l'agent — configure le comportement, les règles et les sources de connaissance |
| Topic | Scénario de conversation prédéfini dans Copilot Studio, déclenché par des mots-clés ou intentions NLU. Chaque topic définit le flux de conversation pour un type de question. | Ex. : Topic « Vacances et Congés » déclenché par les mots « vacances », « congé », « fériés » |
| NLU (Natural Language Understanding) | Capacité d'un système IA à interpréter le sens d'un texte en langage naturel — identifier l'intention et les entités. | Utilisé par Copilot Studio pour classifier les questions des employés et router vers le bon topic |
| Generative Answers | Fonctionnalité de Copilot Studio activant le RAG : connexion automatique à Azure AI Search + génération de réponse via Azure OpenAI GPT-4o. | Le cœur du traitement des questions RH non préfigurées dans les topics spécifiques |
| Adaptive Card | Format de carte interactive JSON utilisé dans Microsoft Teams et Copilot Studio pour afficher des réponses structurées avec boutons, images et liens. | Utilisé pour les messages de bienvenue, les réponses avec citations, les cartes d'escalade |
| Managed Identity | Identité Azure AD attribuée à un service Azure (ex. : AI Search, Power Automate) lui permettant de s'authentifier auprès d'autres services sans gérer de mot de passe ou clé API. | Utilisée pour les appels AI Search → Microsoft Graph API et Power Automate → AI Search |
| Power Automate | Outil de workflows automatisés Microsoft (low-code) — utilisé pour déclencher l'indexation AI Search lors d'un dépôt de document dans SharePoint. | Exécute le flux `PA-RH-IndexOnDocumentChange` à chaque modification dans la bibliothèque Politiques-RH |
| Microsoft Purview | Suite de conformité et gouvernance des données Microsoft. Utilisé pour le DLP, l'audit logging et les politiques de rétention des conversations. | Stocke les logs d'audit des conversations Copilot pendant 12 mois (PIPEDA) |
| ACL (Access Control List) | Liste de permissions définissant quels utilisateurs ou groupes ont accès à une ressource. Dans notre contexte : permissions SharePoint héritées par les chunks dans l'index AI Search. | Base du security trimming — chaque document et chunk hérite des ACL de sa bibliothèque SharePoint |
| PIPEDA | Loi sur la protection des renseignements personnels et les documents électroniques — loi fédérale canadienne encadrant la collecte, l'utilisation et la divulgation de renseignements personnels. | Impose la pseudonymisation des logs, la rétention limitée à 12 mois, le consentement et le droit à l'oubli |
| Power Platform CLI (`pac`) | Interface en ligne de commande pour gérer les solutions Power Platform (Copilot Studio, Power Automate, Dataverse) — export, import, configuration. | Utilisé dans le pipeline Azure DevOps pour automatiser les déploiements |
| HNSW (Hierarchical Navigable Small World) | Algorithme de recherche approximative de plus proches voisins dans un espace vectoriel haute dimension — utilisé par Azure AI Search pour la recherche vectorielle. | Configuré avec m=4, efConstruction=400 dans l'index `rh-documents-index` |
